1.1 Justificación técnica del tipo de base de datos (/2)

Elegir entre base de datos transaccional y/o NoSQL según los requerimientos funcionales y no funcionales.

Presentar el diagrama del modelo de datos con claves primarias, foráneas y restricciones de integridad (CHECK, UNIQUE, reglas personalizadas).

Implementar encriptación a nivel de columna para proteger información sensible (credenciales, identificadores, etc.).

1.2 Validaciones y reglas de negocio (/2)

Replicar en la base de datos las validaciones del front-end mediante restricciones (CHECK, DEFAULT, UNIQUE) y triggers.

Usar procedimientos almacenados para validar operaciones críticas.

Aplicar transacciones (BEGIN TRANSACTION, COMMIT, ROLLBACK) para mantener la integridad ante errores.

1.3 Auditoría de entidades (/2)

Implementar auditoría en al menos tres entidades principales.

Usar una tabla centralizada y triggers para registrar inserciones, actualizaciones y eliminaciones (usuario, fecha, tipo de operación).

1.4 Reportes complejos (/2)

Diseñar al menos dos reportes relevantes.

Utilizar vistas que integren múltiples tablas con funciones, filtros y agregaciones útiles para la toma de decisiones.

1.5 Plan de Seguridad Informática y de Base de Datos (/2)

Detallar políticas de seguridad: sistema operativo, infraestructura (cloud/on-premise), motor de base de datos.

Incluir buenas prácticas: control de acceso, encriptación, respaldos, monitoreo y gestión de vulnerabilidades.
